# Issue Tracker

- **Type:** Local markdown
- **Location:** `.scratch/<feature>/` in this repo
- **External PRs as triage surface:** No

## Workflow

Issues are created as markdown files under `.scratch/<feature>/`. Each issue gets a directory with an `ISSUE.md` file containing the issue's metadata and description.

## Creating Issues

```bash
mkdir -p .scratch/<feature-slug>
```

Then create `.scratch/<feature-slug>/ISSUE.md` with frontmatter and content.
